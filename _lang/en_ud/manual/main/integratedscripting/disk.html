Scripting Disk

Scripting Disks are responsible for storing scripts, and are to be inserted into Scripting Drives. They can be created by combining some Variable Cards with Mendesite.

Once a Scripting Disk has been inserted into a Scripting Drive, they will obtain a unique numerical identifier. This identifier will be used to modify scripts on this disk from within the Scripting Terminal. The identifier is also used to link script elements to Variable Cards.

A Scripting Disk can safely be moved to another Scripting Drive without loss of script data, even when on a different network. Variable Cards that refer to this script's identifier will however start throwing errors when not on the same network anymore.

A Scripting Disk has no limit in terms of its storage size, but splitting up scripts across multiple disks is recommended for management reasons.
