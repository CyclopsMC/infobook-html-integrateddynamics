Процесс крафта

Этот раздел объяснит, как Интерфейсы Крафта обрабатывают процесс крафта. Эта информация необходима для максимального использования мода и устранения любых проблем, которые могут возникнуть.

Когда начинается задание на крафт (например, с помощью Писателя Крафта), используется один (или несколько) Интерфейсов Крафта, имеющих рецепт, соответствующий данному заданию. Интерфейс Крафта отвечает за ввод всех ингредиентов рецепта в связанную машину. Это происходит на основе содержимого сети хранения, предоставленного Интерфейсами Tunnels.

Однако интерфейс Crafting Interface не будет импортировать результаты ремесла из связанной машины. Игрок несет ответственность за то, чтобы результаты ремесла попали обратно в сеть. Исключением является стол ремесленника, для работы с которым требуется только интерфейс Crafting Interface. Интерфейс Crafting Interface будет отслеживать изменения в сети хранения и пометит задание ремесла как выполненное только после того, как в сети хранения появятся ожидаемые результаты. Этого можно добиться, установив на машину активный Integrated Tunnels importer или позволив машине автоматически выводить данные в Crafting Interface. Все (предметы, жидкости, энергия), что вставляется в Crafting Interface, будет вставлено в сеть хранения для настроенного канала.
